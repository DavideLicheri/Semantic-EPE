# AI Context - ECES Project Documentation

Questa cartella contiene documentazione strutturata per istruire agenti AI sul progetto ECES (EURING Code Exchange System).

## Scopo

Fornire un "frammento di realtà" completo e accurato che permetta a un agente AI di:
- Comprendere l'architettura del sistema
- Conoscere le decisioni tecniche prese
- Capire il modello dati e le API
- Sapere come deployare e manutenere il sistema
- Conoscere i problemi risolti e le soluzioni implementate

## Struttura Documentazione

1. **01-PROJECT-OVERVIEW.md** - Panoramica generale del progetto
2. **02-ARCHITECTURE.md** - Architettura tecnica del sistema
3. **03-DATA-MODEL.md** - Modello dati e strutture EURING
4. **04-API-ENDPOINTS.md** - Documentazione API REST
5. **05-FRONTEND-COMPONENTS.md** - Componenti React e UI
6. **06-AUTH-SYSTEM.md** - Sistema di autenticazione e autorizzazione
7. **07-DEPLOYMENT.md** - Procedure di deployment su ISPRA
8. **08-DESIGN-DECISIONS.md** - Decisioni tecniche e motivazioni
9. **09-FILE-MAP.md** - Mappa dei file principali del progetto
10. **10-KNOWN-ISSUES.md** - Problemi noti e soluzioni

## Come Usare Questa Documentazione

Quando inizi a lavorare sul progetto ECES:
1. Leggi prima `01-PROJECT-OVERVIEW.md` per il contesto generale
2. Consulta `02-ARCHITECTURE.md` per capire come è strutturato il sistema
3. Usa `09-FILE-MAP.md` per orientarti nel codice
4. Riferisciti agli altri documenti per approfondimenti specifici
